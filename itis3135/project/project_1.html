Project 1

Overview

The purpose of this website is to allow users of the site to view artwork from my client and be able to submit commissions/ requests for a piece of art. The website works as both a showcase of my clients artwork and a piece for custom artwork for money.

Intended users of the website are anyone who wants to see more artwork from my client or request custom artwork.

Page design

Images

  • Purpose || To show users the clients art as-well-as give the user an idea of the types of artwork they can commission. Images will also viewable through google images; directing traffic to the clients website
  • Audience || The intended audience for the most part is anyone
  • Content || The content for the page will be images of art both commissioned and original.
  • User data entry || None needed for this page
  • Buttons || A fullscreen button for the image
  • Actions || Allow the user to view the image in fullscreen

Contact

  • Purpose || Users can see the clients contact info like email, as-well-as all the social media the clients has.
  • Audience || Users who want to contact for any reason that my not be a artwork request.
  • Content || Will be a list of all the ways in which a user can contact the client
  • User data entry || None need for this webpage
  • Buttons || The buttons/ hyperlinks will be for the social media content
  • Actions || The images of the social media content will redirect them to the appropriate website

Requests

  • Purpose || Users can use this page to submit requests to the client. The page will also have what types of requests the user cna make.
  • Audience || Users who are interested in the artstyle of the client and wants to have a custom art piece done for them.
  • Content || The page will have info about requests and a form the user fills out
  • User data entry || The user enters contact info, info on the art work they want requested
  • Buttons || Buttons to submit info to the client
  • Actions || A system which will allow the client to see the users requests

Blog

  • Purpose || For users to see artworks in process or more personlised info my clients wants to share with people
  • Audience || Users who want to see some behind the scene type content.
  • Content || The content will be both images and text, users can like the content as well
  • User data entry || None need for this webpage
  • Buttons || Buttons to like a post on the blog, images can also be viewd in fullscreen
  • Actions || All the images can be viewed in fullscreen when the user clicks on the button

About
